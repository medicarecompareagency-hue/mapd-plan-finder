## What's been done in this branch

### Schema (`prisma/schema.prisma`)
Three enums and six additive columns on `Plan`. Already documented in
previous revision of this file â€” see git history.

### Scripts (`scripts/`)
- `import-cms-data.ts` â€” updated to populate new columns on import
- `backfill-missing-plans.ts` â€” NEW. Adds SNP/PACE/MSA/COST/MA_ONLY
  plans that NBER landscape doesn't publish. Uses `PlanArea.txt` as
  authoritative footprint. Batch size 50, connection-recycling every
  ~2,500 rows to avoid Supabase session-pooler prepared-statement
  exhaustion (fixed a deterministic hang at row 251K in the earlier
  rev).
- `cleanup-nonshoppable-mapd.ts` â€” NEW. One-shot cleanup of MAPD rows
  in the DB whose `(planId, state, county)` isn't in the NBER landscape
  CSV. Used to remove ~220K non-shoppable MAPD rows the earlier hung
  backfill runs left behind.
- `update-plan-category.ts` â€” NEW. Fast retroactive fill of the new
  Plan Type columns on rows imported before those columns existed.
  Reads `pbp_Section_A.txt`, issues ~2,600 `updateMany` calls keyed
  by `planId`. ~17 min for a full year, vs ~3.5 hours for a full
  re-import.
- `fallback-classify.ts` â€” NEW. Classifies rows whose planId isn't in
  `pbp_Section_A.txt` using only their legacy `planType` text. ~75K
  rows (26% of the DB) fell in this bucket because NBER lists plans
  CMS didn't file PBP detail for (carrier withdrawals between CMS and
  NBER snapshots).
- `prepare-pbp.ts` â€” NEW. Standalone download + extract of the PBP
  Benefits ZIP, no DB ops. Lets you stage raw files without triggering
  the slow full-import flow.
- `verify-plan-category.ts` â€” read-only, 15 sanity checks.

## Known data gaps (non-blocking)

- ~3,486 planIds (~26% of rows, across both years) are in NBER
  landscape but NOT in `pbp_Section_A.txt`. These got classified by
  `fallback-classify.ts` from legacy `planType` alone â€” they have
  `planCategory` and `cmsContractType` populated but lack
  `snpSubtype`, `chronicConditions`, `hasPartD`, `isZeroDollarDsnp`.
  All of them classified as MAPD / MSA / COST (no SNP). This is fine
  for the dropdown feature: they're filterable by category, just not
  by SNP sub-filters (which don't apply to them anyway).

- NBER landscape publishes 1-2 years behind the plan year. The
  `import-cms-data.ts` downloader falls back to the prior year if the
  current isn't available, and logs a loud warning. Always run
  `backfill-missing-plans` after `import-cms` to catch the gap.
