# SOB Scraper — Design Doc

Last updated: 2026-04-23
Status: design only, no code shipped yet

## Problem

2,484 unique plans in the DB have no PBP match — every service-level
copay field (emergencyRoomCopay, outpatientHospitalCopay, hospitalStayCopay,
skilledNursingCopay, catScanCopay) is NULL. These plans are searchable
because they exist in the CMS Landscape file, but CMS never published
PBP benefit rows for them in the 2026 extract. Results: search returns
them but the comparison grid shows all N/A.

Source of truth for the missing data: each carrier's Summary of Benefits
(SOB) and Evidence of Coverage (EOC) PDF, published publicly.

## Scope

- Input: `scripts/data/no-pbp-plans.csv` (2,484 plans, generated by
  `scripts/list-no-pbp-plans.js`).
- Output: per-plan copay + benefit fields in the `Plan` table,
  written ONLY where currently NULL (never overwriting PBP data).
- Target fields:
  - emergencyRoomCopay, outpatientHospitalCopay, catScanCopay (Float)
  - hospitalStayCopay, skilledNursingCopay (String, per-day)
  - drugTier1..6Copay (Float), drugTierCoinsuranceMask (String)
  - dentalBenefits, hearingBenefits, visionBenefits, transportationBenefit (String)
  - medicalDeductible, drugDeductible, monthlyPremium, maxOutOfPocket (Float)

## Pilot carrier: Ascension Complete (67 plans)

Chosen because:
- Confirmed no-PBP baseline (handoff flagged H7556-2).
- Manageable pilot size.
- Single carrier = single SOB template to parse.

After Ascension parser proves out, expand to Cigna (134), Wellcare (152),
Aetna (239), UHC (281), Humana (286). Each carrier needs its own parser
because layouts differ.

## Pipeline

Three scripts, run in order. All must be COALESCE-safe (only fill NULLs).

### 1. `scripts/download-sob-pdfs.js`

- Input: `scripts/data/no-pbp-plans.csv` filtered to a carrier.
- Find SOB URL per plan. Two sources to try in order:
  1. CMS Plan Finder JSON API (medicare.gov has `/api/plans/sob/{contract}/{plan}/{year}`).
     Undocumented but used by the Plan Finder UI. Probe and cache.
  2. Carrier website scraper. Ascension Complete publishes SOBs at
     `https://www.ascensioncomplete.com/members/medicare-advantage-plans.html`
     with a per-state/plan picker. Will need a per-carrier URL builder.
- Output: `scripts/data/sob-pdfs/<contract>-<plan>-<year>.pdf`
- `scripts/data/sob-pdfs/` added to .gitignore (done 2026-04-23).
- Throttle: 1 req/sec to avoid getting 429'd. Retry 3x on transient fails.

### 2. `scripts/parse-sob-ascension.js`

- Input: each PDF in `scripts/data/sob-pdfs/` matching Ascension plans.
- Library: `pdfplumber` (Python) via child_process OR `pdf-parse` (Node).
  Start with `pdf-parse` to keep the toolchain in Node; fall back to
  pdfplumber if tables don't parse cleanly.
- Ascension SOB is CMS-standard-ish. Target the "Medical Benefits Chart"
  section. Key rows:
  - "Inpatient hospital coverage" → hospitalStayCopay (per-day string)
  - "Skilled nursing facility" → skilledNursingCopay
  - "Emergency care" → emergencyRoomCopay (Float)
  - "Outpatient hospital services" → outpatientHospitalCopay
  - "Diagnostic radiology services (e.g. MRI, CT scan)" → catScanCopay
  - "Outpatient prescription drugs" table → drugTier1..6Copay
  - "Dental services" → dentalBenefits (string summary)
  - "Hearing services" → hearingBenefits
  - "Vision services" → visionBenefits
  - "Transportation" → transportationBenefit
- Regex per field. Write partial results; don't fail the whole plan if
  one field is ambiguous.
- Output: `scripts/data/parsed-sob-ascension.json` — one record per
  plan, indexed by `<contract>-<plan>-<year>`.

### 3. `scripts/enrich-plan-copays-from-sob.js`

- Input: `scripts/data/parsed-sob-*.json` (all carriers).
- For each plan record:
  ```js
  await prisma.plan.updateMany({
    where: {
      planId: record.planId,
      planYear: record.planYear,
      emergencyRoomCopay: null,  // COALESCE guard
    },
    data: { emergencyRoomCopay: record.emergencyRoomCopay },
  });
  ```
  One `updateMany` per field, each guarded by `<field>: null` so we
  never overwrite PBP data.
- Log fill counts per field. Target: >80% fill rate on pilot carrier
  before generalizing.

## Risks

- Layout drift between plan years (AEP = new SOBs every October). Parser
  will need a 2027 refresh. Keep regexes loose and well-commented.
- Some carriers serve SOBs behind JS-rendered pages. If Ascension
  requires JS, fall back to Puppeteer/Playwright for the downloader.
- ~25% of SOBs may have benefit rows that aren't parseable by regex
  (carrier-specific wording). Plan for a manual-entry escape hatch:
  any plan still NULL after scrape gets flagged for human review.
